Basic

  • git config --global user.name blabla – ім’я, яке буде відображатися при створенні комітів
  • git config --global user.email blabal@bla - ім'я яке буде видиме при коміті в репозиторій
  • git init - ініціалізує і запускає
  • git status - показує стан фалів в робочому каталозі
  • git add - Команда git add додає вміст робочого каталогу в індекс (staging area) для подальшого комміту.
  • git commit - це команда Git для запису індексованих змін в репозиторій.
  • git log - перераховує коммітов, зроблені в репозиторії в зворотному до хронологічного порядку - останні коммітов знаходяться вгорі.
  • git checkout - переходить на потрібну гілку
  • git branch - перераховує ваші гілки, створює нові, видаляє і перейменовує їх.
  • git push - використовується для встановлення зв'язку з віддаленим репозиторієм, обчислення локальних змін відсутніх в ньому
  • git pull - це, команда git fetch, після якої відразу ж слід git merge
  • git merge - використовується для злиття однієї або декількох гілок в поточну.
  • git clone це утиліта командного рядка Git для вибору існуючого сховища та створення його клону
c.gitignore сповіщає git, які файли (або шаблони) слід ігнорувати. Як правило, він використовується для того, щоб уникнути передачі тимчасових файлів з вашого робочого каталогу,
які містяться в інших соавторах, таких як продукти компіляції, створення тимчасових файлів IDE и т.д.

Advanced

  • git revert - з'ясовує які зміни потрібно скасувати, скасовує їх і створює новий Комміт, в якому ці зміни проведені
  • git reset - повертає до потрібного коміту, видаляє коміт, який ще не відправлений у репозиторій
  • git rebase - стискає всі зміни в один «патч». Потім він інтегрує патч в цільову гілку.
  • git cherry-pick -використовується для перенесення окремих коммітов в інше місце, зазвичай між гілками розробки та обслуговування
  • git stash -Команда дозволяє на час «здати в архів» (або відкласти) зміни, зроблені в робочій копії, щоб ви могли застосувати їх пізніше.
